# logger（日志工具）测试方案

## 1. 目的与覆盖范围

`tests/logger` 覆盖日志工具的功能，包括日志级别控制（debug 模式）、日志前缀显示以及从配置创建日志器。

主要覆盖（以测试文件实际引用为准）：

- 日志创建：**createLogger**
- 从配置创建日志器：**createLoggerFromConfig**
- 日志级别：**info、warn、error、debug**

## 2. 用例表（覆盖现有测试）

### 2.1 info

| 序号 | 场景 | 期望 |
|------|------|------|
| 1 | 非调试模式 | 不输出 |
| 2 | 调试模式带前缀 | 输出带前缀的日志 |

## 3. 预期正确输出与潜在错误

- **正确**：日志器在调试模式下输出 info 和 debug 日志，warn 和 error 始终输出；前缀正确显示；从配置创建日志器正确委托。
- **潜在错误原因**：调试模式判断错误导致日志输出不符合预期；前缀格式错误；console spy 未正确恢复导致影响其他测试。
